Builder section

Runs a couple of Klocwork commands, in order to launch the analysis of the source code.
Needs an installation name (see Hudson configuration to set it up, it is also mandatory),
and a project name (which must be an existing project name).

Parser section

Parses the .xml file given by kwinspectreport. You can specify your own .xml file, or let the default value
which means that you must launch the Klocwork build so as to generate a .xml file.
For more informations about the .xml files and the Klocwork commands, please refer to the Klocwork Manuals.
